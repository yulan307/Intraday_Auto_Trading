# 初级开发文档说明
##1. 背景
现在有资金想进行定投，保持每天投入。单买入位置希望处于当天的低位。设计一套当天自动买入系统。

## 2. 策略
-不关注长期趋势，只通过开盘前的技术指标，开盘后30分钟内的价格波动来判断今日的趋势。
-如果趋势向上，则立即下单
-如果趋势非向上，则启动15min追踪机制，在确认反弹后下限价单
-在下单后，若低点持续更新，则取消限价单，返回跟踪模式，直到出现新的反弹确认
-如果当天没能确认反弹，则在最后15min强制买入

## 3.模块与功能
### 3.1 交易日触发
-每个美东交易日开盘30分后触发一次交易逻辑
-如果当天是非交易日，则不触发  
-输入:无
-输出:当前日期，当前时间

### 3.2 标的选定
-根据开盘前后30分钟的交易数据，计算技术指标，期权信息，判断各个标的的趋势
-根据账户本交易周的订单信息，对已买标的和未买标的进行权重赋值
-根据各个标的的趋势和权重，选定当天的交易标的
-输入:当前日期，当前时间，账户订单信息，标的交易数据
-输出:当天的交易标的，买入策略（立即买入/追踪买入）
-数据来源:
--当前日期，当前时间：交易触发模块
--账户订单信息：账户交易模块
--标的池：标的管理模块
--标的交易数据：数据获取接口，包括IBKR实时bar数据和历史数据，Moomoo的期权数据，IBKR的期权数据（现在无权限，后续考虑接入）
--执行逻辑
---根据 当日趋势量化分析.md 文档中定义的技术方法，获取该标的趋势判断
---由于目的是买低点，所以对弱势拖尾赋予高权重，震荡预期赋予中权重，强势赋予低权重
---根据账户订单信息，对已买标的赋予低权重，未买标的赋予高权重，权重值根据订单数量递减
---根据趋势判断和已买标的情况，继续排序，选定当天的交易标的和买入策略

### 3.3 当日交易执行
-根据选定的交易标的和买入策略，执行当天的交易逻辑
-如果买入策略是立即买入，则直接下单
-如果买入策略是追踪买入，则启动15min追踪机制

### 3.4 15min追踪机制
-获取开盘后所有15min bar数据，确定当前最低close价格
-每15min更新一次最低close价格
-如果连续N个15min bar的都没有更新最低close价格，则下限价单，价格为当前最低close价格乘以一个系数（例如1.01）
-如果在下单后，出现新的最低close价格，则取消限价单，返回追踪模式。如果已经完成订单，退出当日交易。

### 3.5 最后15min强制买入
-如果在开盘后最后15min之前都没有确认反弹，则在最后15min强制买入，价格为当前最低close价格乘以一个系数（例如1.01）

## 4. 交易数据来源
- IBKR提供的实时bar数据和历史数据
- Moomoo提供的期权数据
>>新增，将获取到的数据保存到本地db，使用sqlite进行管理，方便后续分析和回测使用。数据表设计包括标的基本信息表、以及所有需要且要保存的交易数据表，例如日内15min bar数据表，期权数据表等。每个数据表都包含必要的字段，例如标的代码、日期时间、开盘价、最高价、最低价、收盘价、成交量等。通过数据获取接口获取到的数据会被自动保存到本地db中，确保数据的完整性和可追溯性。
- 本地sqlite db，数据接口与IBKR和Moomoo的数据接口一致，使得数据获取和存储流程统一，方便后续的数据分析和回测使用。

## 5. 交易执行接口
- IBKR提供的交易接口,在配置文件中提供实际账户或paper账户的选择。与账户管理模块共享账户信息，确保交易执行与账户状态的一致性。
>>新增回测用交易接口，模拟交易执行的逻辑，包括订单的创建、修改、取消等操作。回测过程中，所有的交易执行都会记录到回测用账户db中，确保回测结果的完整性和可追溯性。

## 6. 标的管理
-标的池由用户自定义，是一个固定的标的池

## 7. 账户管理
-通过IBKR的账户接口获取账户信息，包括订单信息和持仓信息。在配置文件中提供实际账户或paper账户的选择。与交易执行接口共享账户信息，确保交易执行与账户状态的一致性。
-在本地的虚拟账户，记录订单信息和持仓信息，和最大投入金额
>>新增回测用账户db，使用sqlite进行管理，包含订单信息表和持仓信息表等必要的字段，例如标的代码、订单类型、数量、价格、订单状态等。回测过程中，所有的交易执行都会记录到这个账户db中，确保回测结果的完整性和可追溯性。每次回测开始时重置账户。

## 8.回测系统
-设计一个回测系统，能够模拟历史数据上的交易执行，评估策略的表现。回测系统会使用本地sqlite db中的历史数据进行模拟交易执行，并记录所有的交易结果到回测用账户db中。回测结果可以包括总收益率、最大回撤、胜率等指标，帮助评估策略的有效性和风险水平。
